Abstract: To address the challenges of subjective grading, untraceable plagiarism and ghostwriting, and tamper-prone evaluation records in smart education scenarios, a blockchain-based smart education academic integrity assurance system named EduChain is proposed. The system adopts a five-phase pipeline architecture of "Assignment Publish–Submission & Stake–AI & Peer Review–Challenge Window–Optimistic/Arbitration Settlement", employing an optimistic execution model that trusts student submissions and review conclusions by default, triggering costly sandbox verification and arbitration only upon academic disputes, thereby significantly reducing on-chain overhead for daily educational administration. A game-theoretic incentive mechanism based on a credit-stake and academic-reputation dual-asset model is designed, with a dynamic stake oracle ensuring honest behavior as the dominant strategy for all participants. A decentralized arbitration protocol combining VRF-based random committee election and EIP-712 structured data signature voting is introduced. A complete prototype system comprising an on-chain protocol layer, an off-chain AI review service layer, and a user interaction layer is implemented to demonstrate engineering feasibility.

Key words: smart education; blockchain data management; optimistic execution; game-theoretic incentives; academic integrity; decentralized arbitration

## 引言

在智慧教育与计算教育学场景中，学生作业（尤其是编程作业）的评价长期面临三类结构性难题[1-2]：评分主观性强，不同评审者对同一作业的评分可能差异显著；抄袭与代写难以追溯，传统的查重系统依赖中心化数据库，容易被绕过或篡改；评价记录易被篡改或丢失，集中式教务系统将所有信任集中于单一机构，既缺乏过程透明性，也难以支撑跨校、跨学期的学术档案互认[3]。

区块链技术的不可篡改、去中心化与可追溯特性为解决上述问题提供了新的技术路径[4-5]。近年来，教育区块链领域的研究主要聚焦于学籍档案管理与学历证书存证[6-7]，但将区块链技术系统性地应用于日常作业评价与学术争议解决的工作尚不多见。部分研究[8]探索了基于智能合约的作业提交系统，但缺乏完整的争议解决机制；另有工作[9]提出基于代币激励的同行评审，但未考虑女巫攻击等博弈论安全问题。

在区块链数据管理范式方面，Optimistic Rollup[10-11]的乐观执行思想提供了重要启发：默认假设交易有效，仅在争议时触发验证。在去中心化争议解决方面，Kleros[12]的去中心化仲裁协议采用 Schelling Point 博弈激励陪审员投票，但其依赖纯主观投票、缺乏客观验证手段。基于大语言模型（large language model, LLM）的代码评审方法[13-14]展现了理解程序语义的潜力，但幻觉问题导致评审结论的可靠性存疑。

本文设计的 EduChain 系统将"乐观默认、异常裁决"的协议结构引入学术评价流程，核心贡献包括：（1）将乐观执行思想创新性地应用于教育评价场景，设计"默认信任、争议触发"的按需验证模型，显著降低日常教务管理开销；（2）构建学分质押-学术声誉双资产博弈论激励机制，通过动态质押预言机使学术诚信成为占优策略；（3）提出 VRF 随机委员会选举与 EIP-712 多签投票相结合的去中心化仲裁协议，将技术验证与共识机制有机结合；（4）实现 LLM 驱动的 AI 评审代理与确定性沙箱重放机制，使学术争议结论可被任何第三方独立复现。

## 1  相关技术与研究现状

### 1.1  教育区块链与学术诚信

区块链在教育领域的应用研究近年来快速发展[6-7]。Blockcerts[15]等项目探索了基于区块链的数字学历证书标准，实现证书的防伪与跨校互认。Sony Global Education[16]开发了基于区块链的学业成绩管理系统。然而，上述工作主要集中于证书与成绩的存证层面，未涉及日常作业评价与学术争议的完整流程。在学术诚信方面，传统抄袭检测工具（如 Turnitin、MOSS）依赖中心化数据库进行代码相似度比对[17]，存在数据库覆盖不全、比对结果不可独立验证等问题。

### 1.2  区块链乐观执行机制

Optimistic Rollup[10-11]是以太坊 Layer 2 扩容方案的核心范式，其设计哲学为"乐观假设正确，争议时再验证"。交易批次提交后默认有效，在挑战期内任何人可提交欺诈证明触发链上验证。该机制将计算从链上转移至链下，仅在争议时产生链上开销。本文将该思想迁移至教育评价场景：默认信任学生提交的作业是原创且合格的，AI 与同行评审的结论是可靠的，仅在出现学术争议（抄袭举报、评分异议）时才触发代价较高的仲裁流程。

### 1.3  去中心化争议解决

Kleros[12]是目前最具代表性的去中心化仲裁协议，采用 Schelling Point 博弈激励陪审员投票趋向真实。其局限在于：陪审员选举基于代币权重而非专业能力，且缺乏客观的技术验证手段。本文通过 VRF[18]实现不可预测的委员会选举，结合确定性沙箱重放为仲裁提供客观依据——验证结论不是主观投票结果，而是对确定性执行结果的客观背书。

## 2  系统设计

### 2.1  系统模型与参与方

EduChain 系统中存在五类参与方：教师（作业任务发布者）、学生（作业提交者）、同行评议者（独立评分人）、举报者（争议提出者）与仲裁者（委员会成员）。除教师外，其余四类均为注册参与者，仅通过链上协议交互。

威胁假设上，系统不依赖任何中心化可信第三方，假设任意参与方均可能为理性自利主体，包括抄袭代写、敷衍评审、串通打分、恶意诬告等；攻击者可通过创建大量低成本身份发起女巫攻击（如刷小号互评）。在此模型下，协议的核心目标是使诚实行为（原创提交、认真评审、据实举报）成为所有参与方的占优策略。

### 2.2  整体架构

EduChain 采用链上协议层与链下执行层分离的双层架构。

**链上协议层（智能合约）：** 注册合约管理学生/教师的去中心化身份注册、学分质押锁定与学术声誉追踪；争议解决合约承载完整的五阶段作业评审生命周期；仲裁委员会合约实现委员会的可验证随机选举与签名聚合验证；质押预言机合约根据可调的教育激励参数动态计算最低质押门槛。

**链下执行层：** AI 评审代理承担作业质量评审、抄袭检测、验证代码生成、仲裁推理等需要语义理解的任务；隔离沙箱服务为查重与功能验证代码提供完全隔离、确定性可重现的执行环境；去中心化存储网络（IPFS）承载完整的作业包、验证证据与评审记录。

### 2.3  五阶段学术评审流水线

流水线状态机由六种状态组成：开放（Open）、已提交（Proposed）、评审中（InReview）、被争议（Challenged）、已最终化（Finalized）、已罚没（Slashed）。

**阶段一：作业发布。** 教师发布作业任务，附带要求哈希、评分标准（硬约束）、争议窗口时长（默认48小时）、最低质押与作为奖励池的学分。任务发布后状态置为开放。

**阶段二：学生提交与质押。** 学生将作业的三棵 Merkle 树根（状态根承载内容哈希、证据根承载测试结果等证据、追踪根承载开发过程轨迹）连同完整作业包的 IPFS 指针提交至链上，同时锁定一笔学分质押作为对原创性的经济担保。

**阶段三：AI 评审与同行互评。** AI 评审代理首先对作业进行多维自动评审（正确性、代码质量、文档、原创性）。同时多位同行评议者采用提交-揭示两阶段机制独立打分：提交阶段上链分数哈希，揭示阶段公开分数与盐值，防止"跟随串谋"。

**阶段四：争议窗口。** 任何注册参与者（提交者本人除外）均可在窗口期内发起争议。争议的核心载荷不是文字争辩，而是一份可执行的验证代码——例如证明代码与公开来源高度相似的查重对比，或证明功能不符合声明的功能测试，并附以举报者质押。

**阶段五甲：乐观结算路径。** 若窗口期无人争议且同行评分均值达到通过阈值，协议进入乐观结算：学生取回质押并获得学分奖励、声誉+10；同行评议者按 Shapley 值近似分配评审奖励。全过程仅消耗极低链上开销。

**阶段五乙：仲裁结算路径。** 若发起争议，协议进入仲裁路径：VRF 从声誉 ≥ 200 的活跃参与者中随机选出仲裁委员会 → 委员在隔离沙箱中独立运行举报者的验证代码 → 对判定结果进行 EIP-712 签名投票（67% 法定人数）→ 链上原子执行学分罚没与声誉调整。

### 2.4  博弈论激励机制

将参与者交互建模为不完全信息博弈。学生策略集 Σs = {诚实提交, 作弊}，举报者策略集 Σc = {据实举报, 恶意诬告, 不举报}。设 Ch 为诚实完成作业成本，Cc 为作弊成本，Sp 为学生质押额，Pdetect 为抄袭被检测概率，Parb 为仲裁正确率，α 为罚没学分分配给举报者的比例。

诚实学生期望收益 E[πs_honest] = R - Ch（R 为学分奖励），作弊学生期望收益 E[πs_cheat] = (R - Cc)(1 - Pdetect·Parb) - Pdetect·Parb·Sp。令诚实占优，解得最低质押约束：

Sp > (Ch - Cc) / (Pdetect × Parb)                    (1)

基于式(1)设计链上动态质押预言机，根据系统实时参数自动计算最低质押额。

**定理1** 当质押参数满足式(1)时，（诚实提交，据实举报）构成纳什均衡。

代入参数（Pdetect=0.7, Parb=0.95, Ch=2.0学分, Cc=0.1学分, Sp=2.86学分），作弊学生期望收益为-0.065学分 < 0，恶意举报者期望收益为-0.864学分 < 0，据实举报者期望收益为1.0学分 > 0。诚实行为构成占优策略。

此外，采用质押-声誉双资产模型防止女巫攻击：注册需学分质押+W3C DID身份认证，声誉初始100，仲裁候选需声誉≥200，使女巫攻击经济成本远高于收益。

### 2.5  AI 评审代理

设计 LLM 驱动的自主评审代理，依次完成作业用途识别、正确性检查、代码质量评估、抄袭指标检测与评分标准对照。审计深度依据激励动态调整：作业分值高时增加分析轮次与搜索遍数，分值低时降低强度，但对硬性要求的核验始终保留为质量下限。

举报者代理在定位问题后，通过"生成—编译—修复"闭环将问题翻译为可执行验证测试：生成初版代码 → 编译捕获错误 → 回传 LLM 修复 → 循环直至通过，使产出从"可读"升级为"可验证"。

## 3  系统实现与原型验证

### 3.1  原型系统架构

EduChain 原型系统采用分层解耦架构，自底向上划分为链上协议层、链下服务层与用户交互层三个层次。

链上协议层基于 Solidity 智能合约实现，包含四个功能模块：Registry 合约负责参与者的去中心化身份注册、学分质押托管与学术声誉维护；DisputeResolution 合约实现从作业发布到最终结算的完整状态机；ArbitrationCommittee 合约封装 VRF 随机选举与 EIP-712 多签投票的链上验证逻辑；StakeOracle 合约根据博弈论模型动态计算最低质押阈值。

链下服务层以 FastAPI 为框架，封装 AI 评审、代码查重验证、沙箱执行与仲裁辅助评估等核心服务。AI 评审代理由 LLM 驱动，通过编译器在环架构确保生成的验证代码具备可编译性与可执行性。

用户交互层基于 React + TypeScript 实现响应式 Web 界面，通过以太坊 JSON-RPC 协议与链上合约交互，将协议的复杂交互逻辑抽象为直观的图形化操作流程。

### 3.2  原型验证

通过端到端的学术争议场景验证系统各层协作的正确性与完整性。验证流程覆盖 EduChain 五阶段流水线的全部路径：教师发布作业与链上锚定、学生提交语义承诺与质押、AI 自动评审与同行提交-揭示评分、举报者提交抄袭验证证据、VRF 委员会选举与沙箱重放仲裁、经济结算与声誉更新。

验证结果表明：链上协议层正确执行状态转换与经济结算逻辑；链下服务层能够完成代码语义分析并生成可编译的验证测试；用户交互层完整呈现协议各阶段的操作入口与状态反馈。原型系统的成功运行验证了 EduChain 系统设计的工程可行性与各组件间接口的正确性。

## 4  结论与展望

针对智慧教育场景中学生作业评价面临的主观性强、抄袭难以追溯、评价记录易被篡改等问题，提出了基于区块链的智慧教育学术诚信保障系统 EduChain。该系统的核心贡献在于：将乐观执行思想创新性地应用于教育评价场景，实现按需验证的高效机制；构建学分质押-学术声誉双资产博弈论模型，从理论上证明诚实行为构成纳什均衡；设计基于 VRF 与结构化签名投票的去中心化仲裁协议，以可执行验证替代主观投票；实现 LLM 驱动的 AI 评审代理与确定性沙箱重放机制，使争议结论可被独立复现。

当前工作的局限在于：原型系统尚未在大规模真实教学场景中验证性能与鲁棒性；沙箱验证依赖验证代码的可编译性，对非代码类作业（如论文、报告）的适用性受限；区块链交易成本可能影响系统在资源受限环境中的部署。未来将在更大规模教学数据集上进行定量实验验证，扩展系统对多模态作业类型的支持，探索基于零知识证明的隐私保护评审方案，以及面向 Layer 2 网络的低成本部署方案。
